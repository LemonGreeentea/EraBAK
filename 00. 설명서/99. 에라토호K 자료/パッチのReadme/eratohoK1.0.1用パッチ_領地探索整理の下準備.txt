#영지탐색 정리의 사전 준비 2017/04/17

	 내용
		호출 방법에 대해서는 여러가지 구상이 있는 것은 아닌지라고 생각해,
		어떤 처리라도 우선 필요한 것 같아, 양이 많이 시간이 들 것 같은 사전 준비에 대해
		끝마치는 것을 염두에 작업해 보았습니다.

		함수명 그 외, 모두 자유롭게 개변해 주세요.
		제작자님, 영지탐색 제작자의 여러분에게 감사.

	 상세
		+SELECTCASE문으로부터 이벤트 내 1 CASE를 1함수로 변경했습니다.
		+이벤트 종료시에 RETURN 1을 돌려주도록(듯이)했습니다.
		+이벤트를 별파일에 정리했습니다.
		+코멘트 아웃 되고 있던 이벤트는 별파일에 이동. 연번을 제외했습니다.
		+코멘트 아웃 되고 있던 이벤트는 발생하는 이벤트와 같은 형태로 해, 코멘트 아웃 다시 했습니다.
		+남은 발생하는 이벤트의 CASE 번호를 연번으로 다시 거절했습니다. 그런데도 222종류로 대량.
		+각 함수에 필요한 DIM 선언을 추가했습니다.
		+【조교 SLG】eratoho 종합 스레【매각 금액 229】>>636 >>781【조교 SLG】eratoho 종합 스레【호감도 230】>>824 근처
			주의가 있었다{}나@", 눈치챈 곳을 수정했습니다.
		+본체로부터 TRYCCALLFORM로 호출하도록(듯이)했습니다.
			RETURN 0의 경우는 함수가 발견되지 않았던 경우와 동일 취급으로 했습니다.

	 DIM 선언 추가에 대해 비고
		 #「대금업자와 농민」동료의 평가가 오르는 처리에 대해
			+이하의 처리에 대해, 변수 「동료」의 선출 처리가 존재하지 않았기 때문에, DIM 선언을 추가하지 않았습니다.
				IF 동료 != 0 && 대상 == MASTER
					CFLAG:동료:호감도 += 100
					CALL COLORPRINT(@"%ANAME(동료)%의 평가가 올랐다", 칼라_주의, "W")
				ENDIF

			+후보 캐릭터의 선출을 추가할까하고 생각한 것입니다만…….
				아야맥으로부터 동료 전원의 호감도를 상승하고 싶었기 때문에는이라고 생각해, 외람되오나 이하로 했습니다.
				의도가 달랐을 경우는 고쳐 받을 수 있으면 다행입니다.
				FOR LOCAL:0, 0, CHARANUM
					IF CFLAG:(LOCAL:0):소속 == CFLAG:대상:소속 && ! CFLAG:(LOCAL:0):포로처 && ! IS_ANIMAL(LOCAL:0)
						CFLAG:(LOCAL:0):호감도 += 100
					ENDIF
				NEXT
				CALL COLORPRINT(@"%ANAME(대상)%에 대한 동료의 평가가 올랐다", 칼라_주의, "W")
